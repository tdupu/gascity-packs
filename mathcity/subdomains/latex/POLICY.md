Governs how LaTeX work enters and moves through the bead system: what LaTeX
work is a bead, how a LaTeX bead's lifecycle stage is expressed, what the
quality gates prove (and pointedly do not prove), how documents are atomized
into beads, how LaTeX couples to LMFDB records and Magma computations, when
section merges are legal, how compile failures become MREs, and which bead
shapes are anti-patterns. Every rule has an ID and a pass/fail criterion a
skill can mechanically check.

**Rule prefix: LX** (reserved in
[`mathcity/docs/rule-prefix-registry.md`](../../docs/rule-prefix-registry.md)
per PP5.2).

**Division of labor with existing policy.** This document does NOT restate the
brief-pipeline LaTeX rules or the gate charter — it composes them:

- **L1–L4** (`subdomains/brief-system/POLICY.md`) own the *brief-pipeline*
  expression: when gate G6 fires, what satisfies it, and that `notes.tex` is
  never a no-brainer. On any conflict, L-rules win for brief behavior.
- **he-jwmy** (hecke bead; mirrored at
  `~/repos/agent-skills/docs/brief-bundle/policies/latex-gate-policy.md`) is
  the canonical LaTeX-gate charter: the 7 required merge artifacts, the
  covered-file scope definition, and the author-vs-reviewer separation. This
  policy cites it rather than duplicating it.
- **BP-rules** (`mathcity/BEADPOLICY.md`) own bead typing (BP1), research-bead
  protection (BP2/BP5), the bead-side well-formedness of LaTeX beads (BP7:
  target/coverage declaration, gate-evidence acceptance criteria,
  atomize-vs-bundle), and mathematical claim beads (BP8). Per BP7's scope
  note: **on conflict, LX wins for LaTeX workflow mechanics; BP wins for bead
  lifecycle.**
- **LX-rules** (this file) own everything bead-side: bead shape, stages,
  atomization, computation coupling, merge discipline, MREs, anti-patterns.

**Covered file** (used throughout, per he-jwmy): a `.tex` file that is named
`notes.tex` at any depth, OR lives under a `latex/notes/` tree, OR is in the
depth-2 `\input`/`\include` closure of such a file. Out of scope: files under
`scratch/`/`tests/`/`fixtures/`/`tmp` paths with no `\label{}` reachable from
the notes-tier reference graph and outside the depth-2 closure; documented
autogenerated files carrying a `% AUTOGENERATED` marker.

## Pillar 1 — What LaTeX work is a bead (LX1.x)

*The bead system tracks adjudicable LaTeX work. Inline tags track point fixes.
Nothing lives in shadow TODO files.*

- **LX1.1 Covered `.tex` changes destined for push/merge are bead-tracked.**
  Any diff to a covered file that will be pushed or merged has a bead, and the
  `check-latex` evidence for that diff is produced with `--bead <id>` naming
  that bead (evidence lands under `~/gt/tmp-for-taylor/<bead>/`).
  Pass: every covered diff presented for adjudication has a named bead and a
  bead-addressed evidence dir. Fail: a covered diff screened as `unbeaded`, or
  presented with no bead at all → **revise** (file the bead, re-screen).
- **LX1.2 Inline unfinished tags are single-session point markers.** The
  unfinished tags `check-latex` scans for (`\taylor`, `\david`, `\claude`,
  `\note`, `\todo`) mark point work inside the document. A tag whose work
  survives the session that created it, or that needs more than one sitting,
  gets a bead; the tag body is then updated to carry the bead ID (e.g.
  `\todo{he-xxxx: reconcile with §3.2}`). Pass: every unfinished tag older
  than its creating session names a bead. Fail: multi-session work tracked
  only by an anonymous inline tag → **revise**.
- **LX1.3 No shadow TODO surfaces.** LaTeX task state never lives in
  `TODO.md`, a `% TODO list` comment block, a side `todo.tex`, or a notebook
  cell — beads are the tracker (repo rule: `bd` for ALL task tracking).
  Fail: any file or comment block functioning as a LaTeX task list → **revise**
  (convert entries to beads, delete the list).
- **LX1.4 Out-of-scope `.tex` is unbeaded by default; promotion is a bead.**
  Scratch/fixture `.tex` (he-jwmy out-of-scope) needs no bead. The moment such
  a file is promoted into the notes tier (moved under `latex/notes/`, given a
  reachable `\label{}`, or `\input`'d within depth 2), the promotion itself is
  a bead and LX1.1 applies from that diff onward. Fail: a promotion diff with
  no bead → **revise**.

## Pillar 2 — Stage taxonomy (LX2.x)

*Stages are labels on real bd types — never invented types (P5.3 / BP1.1).*

- **LX2.1 Four stages, expressed as labels.** A LaTeX bead carries exactly one
  of the stage labels:
  - `latex-draft` — content being written; compile may fail.
  - `latex-compilation-ready` — compiles clean; labels/refs clean; ready for
    gate screening.
  - `latex-submitted` — the specific diff passed the full gate (LX3.4/L3) and
    the content shipped to its target (merged to main notes.tex, submitted to
    arXiv/journal, or pushed to an LMFDB knowl).
  - `latex-published` — target accepted/published; permanent reference.
  The bd **type** stays in the documented set: default `task` (section-scale
  work), `feature` (a new paper/deliverable), `epic` (document-scale
  migrations, e.g. he-ce4.3), `chore` (typo batches, LX4.5). `bd create -t
  draft` or any invented type → **fail** (P5.3). LaTeX beads additionally
  carry the `[LATEX]` domain label (BP7 convention) so bead-hygiene sweeps
  (`check-math-bead-hygiene`) can find them. Pass: `bd show <id>` shows a
  documented type, the `[LATEX]` label, plus exactly one `latex-*` stage
  label.
- **LX2.2 Stage transitions have mechanical entry criteria.**
  - `→ latex-compilation-ready`: `check-latex` report shows `compile.status`
    of `pass` (or `pass-with-undefined-refs` with a one-line reason), AND
    `check-labels-and-refs` verdict `PASS`.
  - `→ latex-submitted`: full G6 evidence per L3 (compile evidence AND
    reviewer verdict), Taylor's per-diff approval per LX3.4, and the
    submission target recorded in the bead (branch/merge sha, arXiv ID,
    journal, or knowl ID).
  - `→ latex-published`: the permanent identifier (DOI, arXiv version, LMFDB
    knowl ID) recorded in the bead; the bead then enters research-journal
    protection per BP2.2 (`[RESEARCH_JOURNAL]` label + protective defer) —
    published mathematics is a permanent reference and is never destructively
    closed (B3.7).
  A stage label advanced without its entry evidence in the bead → **revise**.
- **LX2.3 Stages are monotone per bead.** A bead never moves backward. A
  regression (bounced referee report, reverted merge, knowl rejected) spawns a
  NEW bead linked to the old one (`discovered-from`), starting at
  `latex-draft`. Removing or downgrading a stage label in place → **fail**.
- **LX2.4 Exactly one stage label at a time.** Zero or ≥2 `latex-*` stage
  labels on a LaTeX bead → **revise** (mechanical: `bd show` label list).

## Pillar 3 — Quality gates (LX3.x)

*check-latex produces evidence, not verdicts. Know exactly what the evidence
proves.*

- **LX3.1 `check-latex` is the mechanical evidence floor.** What it enforces:
  (a) a real compile attempt (latexmk → pdflatex → xelatex → lualatex →
  tectonic), degrading to `toolchain-unavailable` — never a faked result;
  (b) a semantic diff summary (sections, theorem-class environments,
  equations, labels, citations, unfinished tags touched); (c) the evidence
  block `check-latex-report.{json,md}` + `tex.diff` + `compile.log` under
  `~/gt/tmp-for-taylor/<bead>/`. A covered diff adjudicated without this
  evidence block → **fail** (the `latex-gate` formula fails closed on exactly
  this).
- **LX3.2 What `check-latex` does NOT establish — and no bead may claim it
  does.** The scan is grep-based: no macro expansion, no `%`-comment
  stripping, single-pass compile, no PDF-level diff. Therefore a `check-latex`
  pass is NEVER evidence of: mathematical correctness, semantic equivalence
  ("losslessness"), or rendered-output identity. A bead, brief, or close
  reason citing a `check-latex` pass as proof of any of these → **revise**
  (reclassify the claim; produce the he-jwmy item 1–6 artifacts if
  losslessness is actually claimed).
- **LX3.3 Author ≠ reviewer for equivalence claims** (restates he-jwmy
  item 6). The agent that produced a LaTeX diff cannot be the sole author of
  its equivalence/lossless evidence. Independent compile + PDF-diff comes from
  Mayor or a peer distinct from the diff author; the diff author escalates
  rather than self-attests. Self-attested losslessness → **auto-reject** at
  the brief gate (the he-d4l failure pattern).
- **LX3.4 Taylor approves the specific diff, fail-closed** (restates he-jwmy
  item 7 / gate G6 / L4). A covered diff may be pushed/merged only when
  `latex-approval.toml` names Taylor and its `approved_diff_sha` matches the
  sha of the `tex.diff` under evaluation. Stale or blanket approvals do not
  transfer. Mechanical check: `latex-gate-approval-required.sh`.
- **LX3.5 Review workflow: Taylor compiles, agents never pre-compile for
  review.** Review deliveries follow the he-jwmy addendum: source files + a
  README under `~/gt/tmp-for-taylor/<review-name>/`; no pdflatex run on
  Taylor's behalf for the review copy. (The `check-latex` gate compile —
  LX3.1 — is evidence generation, not the review copy; both rules hold.)

## Pillar 4 — Atomization (LX4.x)

*The unit of LaTeX work is one adjudicable diff — reviewable by Taylor in one
sitting. The he-d4l monolith (633/97-line restyle) is the canonical failure.*

- **LX4.1 Default atom: one section of one document.** A LaTeX task bead
  scopes one section (or one contiguous section range) of one root document.
  Pass: the bead body names the root file and the section scope; the resulting
  `tex.diff` stays within that scope. A diff sprawling across unscoped
  sections → **revise** (split).
- **LX4.2 Document-scale work is an epic decomposed per section.** Migrations,
  restructures, and restyles spanning a document are `epic` beads decomposed
  into per-section child tasks BEFORE work starts (pattern: he-ce4.3,
  modular-symbols migration). Starting a document-scale diff under a single
  task bead → **revise**.
- **LX4.3 Per-claim beads only when a claim gates other work.** A single
  theorem/lemma/definition gets its own bead exactly when other beads depend
  on it (the dependency edge must be real: `bd dep`). Tracked claims use the
  `[MATH_CLAIM]` shape (BP8.1) and the writeup bead dep-links the claim bead
  (BP7.4) so `claim_status` can advance to `written-up` when the diff lands.
  Otherwise claims ride with their section. Speculative per-claim beads with
  no dependents → **revise** (fold into the section bead).
- **LX4.4 Figures and tables ride with their section.** No per-figure beads —
  unless the figure/table is generated from computation, in which case Pillar
  8 applies and the generator gets the bead.
- **LX4.5 No per-typo beads; batch cosmetic fixes.** Typos, spacing, and
  pure-cosmetic fixes batch into one `chore` bead per document per pass. A
  swarm of one-line cosmetic beads → **revise** (anti-clog; matches the
  "don't clog the gas-city pipeline" phasing directive).
- **LX4.6 Adjudicability cap.** If the projected `tex.diff` for a bead exceeds
  what Taylor can review in one sitting (guideline: > ~200 changed lines or
  > ~3 sections), the bead must be split before screening. Presenting an
  over-cap diff for adjudication → **revise** (this is the he-d4l lesson,
  stated as a pre-commitment rather than a post-mortem).
- **LX4.7 Atomization floor** (= BP7.3 bundling condition, restated for the
  workflow side). Never split so far that an intermediate bead's midpoint
  state has dangling refs or a non-compiling document — every bead in a
  decomposition must be able to pass its own LX2.2 compilation-ready check.
  And never mix coverage classes in one bead: covered and uncovered `.tex`
  edits split so the gate scope of each bead is uniform. A decomposition
  whose members can't individually compile, or that mixes coverage classes,
  → **revise** (re-cut the seams).

## Pillar 5 — LaTeX-to-LMFDB coupling (LX5.x)

*Mathematical statements that downstream databases cite have two lives — prose
and record. The coupling is explicit or it is drift.*

- **LX5.1 Knowl-destined content links an lmfdb bead.** When covered LaTeX
  contains a definition/statement destined for an LMFDB knowl or a labeled
  database object, G6 fires (per L1(b)–(c)) AND the LaTeX bead links a
  companion bead for the database side (executed with the `mathcity-lmfdb`
  pipeline skills: `lmfdb-object-to-database`, `create-lmfdb-type`,
  `plan-an-lmfdb-webpage`, etc.). Pass: `bd show` on the LaTeX bead lists the
  lmfdb bead as related/dependent. Knowl-destined prose with no linked
  database bead → **revise**.
- **LX5.2 LMFDB labels quoted in prose are verified before close.** Any LMFDB
  label cited in covered `.tex` (e.g. `2.7.m7.28...`) is verified against the
  live database (`search-lmfdb`) before the bead closes; the verification
  (date + query) is recorded in the bead. An unverified label at close →
  **revise**.
- **LX5.3 One direction of truth per statement.** For any statement that
  exists both as prose and as a database record, the bead declares the
  direction: either (a) the record generates the prose (autogenerated `.tex`,
  LX8.2 — fix the generator, never the output), or (b) the prose is
  hand-written and the database sync bead depends on the LaTeX bead. Editing
  both sides by hand in the same bead with no declared direction → **fail**
  (guaranteed divergence).

## Pillar 6 — Merge policy (LX6.x)

*Section merges reorder the reference graph — the most label-dangerous edit
class there is.*

- **LX6.1 `merge-latex-sections` is ON HOLD (gsp-fby) — do not invoke.** The
  skill is a placeholder; its F2 implementation is deferred until F1 (the
  latex-hurdle formula) completes. Invoking it for real work before the F2
  bead opens and closes → **fail**. Until then, merges are manual under
  LX6.2–LX6.3.
- **LX6.2 A merge/reorder is its own bead with pre- and post- label
  screening.** Any diff that moves, merges, splits, or reorders sections runs
  `check-labels-and-refs` on both the merge-base and the branch HEAD; both
  reports land in the bead's evidence dir; the post-report verdict must be
  `PASS` with no NEW orphan labels/refs relative to the pre-report. Then the
  standard stop gate (LX3.4) applies. Missing either report → **revise**.
- **LX6.3 Merge beads are pure-move.** The semantic diff for a merge bead
  shows zero content change to theorem-class environments and equations —
  moves only. Content edits discovered mid-merge go to a separate bead.
  A merge diff that also edits content → **revise** (split; this keeps the
  merge adjudicable as "same content, new order").

## Pillar 7 — Compile-failure MREs (LX7.x)

*A compile error that outlives its session gets the check-mre treatment,
adapted to TeX.*

- **LX7.1 Blocking compile failures not fixed in-session get an MRE.** A
  minimal `.tex` reproducing the error, named `mre-<bead-id>-<slug>.tex`,
  placed under `scratch/` (deliberately out of gate scope per he-jwmy), with a
  header comment block containing:
  ```
  % ERROR:    <exact first error line from the log>
  % TOOL:     <engine + version, e.g. pdflatex (TeX Live 2025)>
  % AUTHOR:   <agent/session id or initials>
  % LAST RUN: <date + machine>
  % STATUS:   FAILS while bug present | CONFIRMED FIXED <date>
  % BEAD:     <bead id of the blocked work>
  ```
  A compile-blocked bead handed off with no MRE and no in-bead reproduction
  recipe → **revise**. (Discipline mirrors `check-mre`: naming, header block,
  polarity, committed — see that skill for the pattern.)
- **LX7.2 MREs stay out of the notes tier by construction.** An MRE must not
  be `\input` by any notes-tier file and must not carry a `\label{}` reachable
  from the notes reference graph — otherwise it silently becomes a covered
  file and the gate fires on scratch. Mechanical check: the he-jwmy
  out-of-scope test passes on the MRE path. Violation → **fail** (move it).
- **LX7.3 MRE polarity flips when the fix lands.** The fixing bead references
  the MRE; on fix, the MRE's `STATUS` line flips to `CONFIRMED FIXED <date>`
  (or the file is retired in the same commit with the close reason noting it).
  A fixed bug whose MRE still says `FAILS` → **revise** (stale corpus).

## Pillar 8 — Computation dependencies (LX8.x)

*Numbers in prose are downstream of computations. The dependency is a bd edge,
not a vibe.*

- **LX8.1 Quoted results trace to artifacts.** Every computational result
  quoted in covered `.tex` (eigenvalue tables, ranks, conductor lists,
  timings) names its source in the bead or in a `.tex` comment adjacent to the
  quote: a `DATA/` path, a `test-*.mag` script + commit, or an LMFDB label
  (then LX5.2 applies). An untraceable number at gate time → **revise**.
- **LX8.2 Autogenerated `.tex` declares itself and is never hand-edited.**
  Generated files carry `% AUTOGENERATED by <generator> — do not hand-edit`
  and the generator is documented in the repo (this is what keeps them out of
  gate scope per he-jwmy). A hand edit to an autogenerated file → **fail**
  (fix the generator, regenerate). A generated file missing the marker is
  covered by the gate like any other file — no marker, no exemption.
- **LX8.3 LaTeX beads block on computation beads via real dependency edges.**
  A LaTeX bead that quotes results from an open Magma computation bead
  carries `bd dep` on it — never a prose note "waiting on the computation".
  Pending values in the document are marked with an unfinished tag naming the
  blocking bead (`\todo{blocked-by he-xxxx: value pending}`), satisfying
  LX1.2. Pass: `bd show` lists the edge. A quoted-but-pending value with no
  edge → **revise**.
- **LX8.4 Recomputes invalidate prose — file the follow-up in the same
  convoy.** When a repair/recompute (S-rule territory: `queue-repairs.sh`,
  `RECOMPUTE_IQ`, etc.) changes a value quoted in covered `.tex`, the repair
  convoy includes a LaTeX follow-up bead to reconcile the prose. A repair
  convoy that changes quoted values with no LaTeX bead → **revise** (P4.3
  aggregate-pass analogue for prose).
- **LX8.5 The notebook loop checkpoints through `.mag`, not through `.tex`.**
  Iterating on the computation behind a table happens in the
  `mag-to-notebook` → Jupyter → `notebook-to-mag` cycle; the `.tex` updates
  only from a committed, versioned `.mag` (or its `DATA/` output). Pasting
  results into `.tex` from an uncommitted notebook → **revise** (LX8.1
  becomes unsatisfiable — there is no artifact to trace to).

## Pillar 9 — Anti-patterns (LX9.x)

*Named bead shapes that are rejected on sight. Each cites the rule it
violates.*

- **LX9.1 The monolith.** One bead, whole-document diff, presented for a
  single adjudication (he-d4l). Violates LX4.2/LX4.6 → **revise** (decompose
  into an epic).
- **LX9.2 No compilation target.** A LaTeX bead that never names its root
  `.tex` file — the file `check-latex` would be run against. Beads touching
  `\input` fragments name the root document, not just the fragment. Missing
  target → **revise** (the evidence engine has nothing to compile).
- **LX9.3 No owner past draft.** Any bead at `latex-compilation-ready` or
  beyond with no assignee. Stage transitions are actions; actions have owners.
  → **revise** (assign or demote is not allowed — LX2.3 — so assign).
- **LX9.4 Self-attested losslessness.** "No information loss, see my own
  report." Violates LX3.3/he-jwmy item 6 → **auto-reject**.
- **LX9.5 The mixed-concern diff.** Content edits + restyle + restructure in
  one bead, so no single verdict can be rendered. Violates LX4.1/LX6.3/B1.2
  (one decision per brief) → **revise** (split by concern).
- **LX9.6 Bead-as-journal.** Using a work bead's notes as the mathematical
  record instead of the `[MATH_RESEARCH]` → `[RESEARCH_JOURNAL]` flow
  (BP2.1–BP2.2), or destructively closing a bead whose notes ARE the record
  (B3.7). → **revise** (relabel and protect).
- **LX9.7 The unbeaded gate dodge.** Screening a covered diff as `unbeaded`,
  or splitting a covered edit across "scratch" files to duck the gate.
  Violates LX1.1/LX1.4 and the fail-closed intent of `latex-gate.toml` →
  **fail**.

## Non-negotiables (quick checklist)

- Covered `.tex` diff headed for push/merge ⇒ a bead + bead-addressed
  `check-latex` evidence (LX1.1); never screened `unbeaded` (LX9.7).
- Stages are labels (`latex-draft` → `latex-compilation-ready` →
  `latex-submitted` → `latex-published`) on REAL bd types — never invented
  types (LX2.1, P5.3); one stage label, monotone (LX2.3–LX2.4).
- `check-latex` pass ≠ mathematical correctness ≠ losslessness (LX3.2);
  equivalence evidence is never author-only (LX3.3); Taylor approves the
  specific diff sha, fail-closed (LX3.4).
- Atom = one section of one document (LX4.1); document-scale work is an epic
  decomposed first (LX4.2); > ~200 changed lines or > ~3 sections ⇒ split
  before screening (LX4.6).
- Knowl-destined prose links an lmfdb bead (LX5.1); quoted LMFDB labels are
  verified before close (LX5.2); one direction of truth per statement (LX5.3).
- `merge-latex-sections` is HOLD — do not invoke (LX6.1); merges are
  pure-move with pre/post label screening (LX6.2–LX6.3).
- Blocking compile failures get a scratch-scoped MRE with header block
  (LX7.1–LX7.2).
- Quoted numbers trace to committed artifacts (LX8.1/LX8.5); computation
  dependencies are `bd dep` edges (LX8.3); autogenerated `.tex` is marked and
  never hand-edited (LX8.2).
- Published-stage beads become protected research journals — never
  destructively closed (LX2.2, B3.7, BP2.2).

## Verdict vocabulary

Reuses the shared brief-cycle vocabulary (POLICY-POLICY) — no parallel
vocabulary:

- **approve** — no LX-rule violations; the bead/diff/audit target is clean.
- **revise** — fixable violations; the verdict names each LX-rule, the
  triggering bead/file, and a one-line remediation.
- **reject** — the approach itself violates a rule with no workaround (e.g.
  self-attested losslessness, a gate dodge). Artifacts only — audits emit
  approve/revise/defer (PP audit rule).
- **defer** — a human call: contested coverage ("is this file notes-tier?"),
  ambiguous direction of truth (LX5.3). Escalate, don't guess.
