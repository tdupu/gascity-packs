# check-latex

The runnable screening artifact behind the **LaTeX HARD GATE**: any edit to
`notes.tex` or any main-branch `.tex` file in the notes-tier of the
`differential-valuations`, `hecke`, `homog`, or `jacobi` rigs requires Taylor's
explicit approval of the **specific diff** before it may be pushed or merged.

This skill produces the evidence Taylor needs to make that call. It does not
adjudicate, does not merge, and does not push. Its output is the artifact the
`latex-gate` formula (F1b) requires and the `he-86wu` gate-release consumes.

Policy lineage: `he-jwmy` (LaTeX-gate-policy — canonical charter), `he-hsoc`
(umbrella spec), `he-86wu` (gate-release criterion, first instance
`polecat/he-d4l`), `he-cnat` (preliminary-skill-first).

## When to use

Use whenever a diff touches a **covered** `.tex` file (see Scope) and you need
the gate evidence before Taylor adjudicates a push/merge. Also used as the
callee of the `latex-gate` formula and to release the `he-86wu` block.

## Scope (covered files) — per he-jwmy

A `.tex` file is **covered** when any of:

- it is named `notes.tex` at any depth, OR
- it lives under a `latex/notes/` tree, OR
- it is `\input`'d or `\include`'d by such a file at depth ≤ 2 (the depth-2 cap
  makes the rule mechanically checkable with
  `grep -E '\\(input|include)\{' notes-tier.tex`).

**Out of scope** (per he-jwmy): a `.tex` file satisfying ALL of (a) under
`scratch/`, `tests/`, `fixtures/`, or a path containing `tmp/scratch/fixture/test`,
AND (b) no `\label{}` reachable from any notes-tier `\ref{}`/`\eqref{}`, AND
(c) not in the depth-2 `\input`/`\include` closure. Autogenerated `.tex` is out
of scope only if the generator is documented AND the file carries a
`% AUTOGENERATED` marker.

## Workflow (3 lines)

1. **Compile** the target with the first available TeX tool
   (latexmk → pdflatex → xelatex → lualatex → tectonic); if none is present,
   report `toolchain-unavailable` and NEVER fake a compile result.
2. **Semantic-diff** the change: enumerate sections, theorem-class environments,
   equations, labels, citations, and unfinished-tag deltas touched.
3. **Emit** the evidence block (`files touched · compile status · semantic
   summary · exact diff pointer`) as `check-latex-report.{json,md}` under
   `~/gt/tmp-for-taylor/<bead>/` for Taylor to approve or reject.

## How (invocation)

```bash
# From inside the repo containing the .tex (git diff needs a repo):
skills/check-latex/check-latex.sh <tex-file> \
    [--base <git-ref>]   # merge-base or branch point; default: working-tree vs HEAD
    [--out <dir>]        # default: ~/gt/tmp-for-taylor/<bead>/
    [--bead <id>]        # bead this screening is for; default: unbeaded
```

The helper `check-latex.sh` is the mechanical evidence engine. It:

- picks the first available TeX build tool and records `compile.status` as one of
  `pass`, `pass-with-undefined-refs`, `fail`, or `toolchain-unavailable` — the
  last is the graceful-degradation path (compile skipped, explicitly NOT faked);
- acquires the diff via `git diff [<base>] -- <file>` (or working-tree vs HEAD),
  falling back to whole-file content when the target is not in a git repo;
- scans the added/removed lines for `\section`/`\subsection`, theorem-class
  environments (`theorem`, `proposition`, `lemma`, `corollary`, `definition`,
  `remark`, `example`, `conjecture`, `question`, `claim`), display-math
  environments, `\label{}`, `\cite{}`, and unfinished tags
  (`\taylor`/`\david`/`\claude`/`\note`/`\todo`);
- writes `check-latex-report.json` (machine-readable) and
  `check-latex-report.md` (the human evidence block) plus `compile.log` and
  `tex.diff`.

## Evidence block (what Taylor gets)

The `.md` report is the decision surface. It contains exactly:

1. **Compile status** — `pass` / `pass-with-undefined-refs` / `fail` /
   `toolchain-unavailable`, with the tool used and a log pointer.
2. **Files touched** — the covered `.tex` files changed by this diff.
3. **Semantic diff summary** — sections, theorem-class environments, equation /
   label / citation / unfinished-tag counts touched.
4. **Diff pointer** — the exact `tex.diff` path Taylor can open, plus the diff
   basis (base ref or working-tree).

## Sub-skills (composed when present)

The full notes-tier screening fans out to sibling sub-skills; this umbrella
runs them if they are installed and collates their verdicts into the report:

- `check-latex-style` — prose / notation / theorem-environment style.
- `check-citations` + `clean-citations` — pinpoint-citation policy (page +
  atom), MathSciNet+JabRef bibtex hygiene, broken `\cite{}` detection.
- `check-labels-and-refs` — orphan labels/refs, non-pinpoint cross-refs,
  `\input`/`\include` closure to depth 2.

When a sub-skill is absent, its section is marked `absent` in the report — the
compile + semantic-diff + evidence core still runs (that core is the gate's
minimum floor).

## Constraints (DRY-RUN contract)

- Read-only outside `~/gt/tmp-for-taylor/<bead>/`.
- Never runs `Edit`/`Write`/`git add`/`git commit`/`git push`/`bd update`.
- Never approves, merges, or pushes. Emits evidence; Taylor decides.
- Never fabricates a compile result. Missing toolchain →
  `toolchain-unavailable`, full stop.

## Fixture test (acceptance)

Invoke against the canonical negative example
`polecat/he-d4l:latex/notes/hyperboloid.tex`. Pass-bar (per he-86wu): the
report enumerates the compile status, the touched sections/theorems/equations,
and a diff pointer; sub-skills (when present) flag the three known failures
(`\perp`/`\top` notation, broken citations, non-pinpoint refs). A clean
post-revision branch produces a report with no sub-skill failures.

## Limitations

- Compile is single-pass by default; `pass-with-undefined-refs` flags the
  second-pass cross-ref case rather than resolving it.
- Semantic scan is grep-based (no macro expansion, does not strip `%` comments);
  it is a triage summary, not a proof of equivalence.
- The two-sided compile (merge-base + branch-HEAD) and the latexdiff/diff-pdf
  PDF artifact required by he-jwmy items 1–2 are produced by running the helper
  once per side and pairing the reports; the PDF-diff sub-artifact is a
  downstream sub-skill (`he-ogkp`) not yet folded into this preliminary.
